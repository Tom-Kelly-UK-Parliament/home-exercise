UK Parliament - Product Team Developer Home Exercise

Thanks for taking the time to do our recruitment home exercise

Overview

We'd like you to make a (very!) small person manager app - it doesn't have to be complicated!

Assessment criteria

You should have been provided an assessment criteria for the exercise separately. You should aim to satisfy all the points on it as this will be referenced when reviewing your completed exercise.

Project

We've setup a template project for you. It includes:

  • An Angular frontend embedded in a .Net web application (all UI components should be implemented in Angular)
  • The beginning of an API - A person API controller exists for you, and the proxy.config has been set for the /api route. See home.component.ts for a very basic example of calling the API
  • Services assembly - To contain any services you might require
  • Data assembly - Houses the in-memory database context with simple Person and Department models. The Departments are seeded into the database in application startup.
  • Test assembly - For tests!

Expected completed solution

What should the finished exercise look like?

When you've finished this exercise these instructions should be gone. In their place should be a very simple person management app:

Please remove the home component and add your own components, or if you prefer, use routing to provide a link to go to your person manager app page

The application should:

  • Show a list of people
  • Allow the person to be loaded into an editor which could sit to the side of the list
  • Allow a user to update the person properties and save them
  • Allow a user to add more people to the list

More detailed instructions are below

Detailed instructions

To complete this exercise please modify this solution and do the following:

  • Extend the person model in the data assembly to include these properties:
    • First name
    • Last name
    • Date of birth
    • Department (should be a dropdown allowing one of the seeded Departments to be selected)
    • Anything else you'd like to add!
  • All fields should be mandatory, and validated appropriately
  • Remove / rename the home component and replace it with your own set of semantically meaningful components
  • Display a list of people
  • People in the list can be selected and load into a simple editor
  • An editor is available next to the list - all properties should be editable (apart from Id!)
  • Allow new people to be added to the list
  • Add the ability to save updates to existing people
  • When a user saves a person record, persist this in the in-memory database. (No need for a real relational backend)
  • If you'd like to you could add other functionality, such as the ability to delete a person etc, but this isn't required
  • Please pay consideration to the visual design and accessibility of the finished product
